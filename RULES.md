# Project Rules

- Build UI with component-first architecture.
- Create reusable micro components before feature components.
- Use centralized Tailwind theme colors only.
- Do not use arbitrary Tailwind values.
- Keep every file below 500 lines.
- Use TypeScript types for all component props and data structures.
- Keep code modular, readable, and clean.
- Use Zustand for shared dashboard state.
- Separate data access logic into hooks.
- Prefer composition-friendly reusable components.
